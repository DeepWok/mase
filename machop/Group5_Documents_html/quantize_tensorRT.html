quantize_tensorRT.export_to_onnx_pass(mg, dummy_in, input_generator, onnx_model_path)

The pass is used to export the quantized model stored in mase graph to onnx model.

Args:

mg (Mase Graph): Mase Graph dummy_in (tensor): The dummy input data from the next iteration of the input_generator. Only been used for taking the shape of the input tensor. input_generator (InputGenerator(class)): Input generator. onnx_model_path (str): provide the path to save the onnx model.

Returns:

mg: the Mase Graph with the onnx model path in the meta.

quantize_tensorRT.generate_tensorrt_string_pass(mg, TR_output_path)

The pass is used to transfer the ONNX model into tensorRT engine string, which will be saved in the TR_output_path.

Args:

mg (Mase Graph): The masegraph with the onnx model path in the “meta”. TR_output_path (str): The path to save the tensorRT engine string.

Returns:

mg: The masegraph with the tensorRT engine string in the “meta”.

Returns the configuration for a given name. If the name is not found in the config dictionary, it returns the default configuration.

quantize_tensorRT.graph_iterator_quantize_by_type_tensorRT_name(graph, config: dict)

Iterates over the nodes in the graph and applies quantization based on the type of the node. If the node type is ‘call_module’, a new module is created with TensorRT.

TypeExamples: seq_blocks.2 or Classifier.0

  1. it will use create_new_module_tensorRT or create_new_module_tensorRT_real to create a new quantized module.

  2. it will update the precision and type in meta.parameters[“common”].

quantize_tensorRT.graph_iterator_quantize_by_type_tensorRT_type(graph, config: dict)

Iterates over the nodes in the graph and applies quantization based on the type of the node. If the node type is ‘call_module’, a new module is created with TensorRT.

TypeExamples: Linear and Conv2d

  1. it will use create_new_module_tensorRT or create_new_module_tensorRT_real to create a new quantized module.

  2. it will update the precision and type in meta.parameters[“common”].
